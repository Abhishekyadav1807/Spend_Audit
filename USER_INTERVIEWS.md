# User Interviews

Status: 3 of 3 completed.

The assignment requires three real 10-15 minute conversations with potential users. I am keeping this file explicit instead of inventing notes. Fabricated interviews would be worse than an incomplete draft and would not produce useful product decisions.

## Interview Script

1. What AI tools does your team currently pay for each month?
2. Who notices when AI tool spend increases?
3. Have you ever downgraded or consolidated AI subscriptions? What triggered it?
4. What would make you trust or distrust an automated savings recommendation?
5. Would you share a public anonymized savings report? Why or why not?
6. If this audit showed >$500/month savings, what would you expect the next step to be?

## Interview 1 — Harshit Tripathi, Claude-heavy workflow

- Name/initials: Harshit Tripathi
- Role: Full-stack developer
- Company stage: Company/team usage
- Direct quotes:
  - "Right now we mainly pay for Claude AI because it fits our workflow best for coding, brainstorming, and writing tasks."
  - "The monthly spend is usually around $20-$60 depending on how much we use the API and how many team members are active."
  - "I would trust an automated savings recommendation if it clearly explained the usage data and showed realistic ways to reduce costs without affecting productivity."
  - "I am okay sharing an anonymized report as long as there is no sensitive company or personal information included."
- Most surprising thing: Harshit had tested other tools and changed plans before, but still stayed with Claude because output reliability mattered more than saving a small amount.
- What it changed in the product: This pushed me to keep recommendation language careful. The audit should not say "switch tools" too casually; it should explain the cost trade-off and respect cases where a slightly more expensive tool is still the right choice for productivity.

## Interview 2 — Anurag, frontend developer using coding agents

- Name/initials: Anurag
- Role: Frontend web developer
- Company stage: Company/team usage
- Direct quotes:
  - "Mostly Cursor and Antigravity for coding and day-to-day dev work."
  - "Around $50-$100 a month depending on usage."
  - "Usually we notice when the monthly subscription renews or API usage spikes."
  - "We have switched or cancelled tools before if they did not feel useful enough."
  - "I would trust it if it showed clear usage stats and realistic ways to save money."
  - "Probably yes, as long as everything is anonymized properly."
- Most surprising thing: Anurag framed tool switching around usefulness, not just cost. If a tool does not clearly help day-to-day development, cancellation becomes easy.
- What it changed in the product: Reinforced that the audit should connect spend to usage fit. A recommendation is stronger when it says why a plan/tool is not pulling its weight, not only how much cheaper another option is.

## Interview 3 — Abhishek Dwivedi, mixed AI workflow

- Name/initials: Abhishek Dwivedi
- Role: Full-stack developer
- Company stage: Company/team usage
- Direct quotes:
  - "We mostly use ChatGPT, Gemini, and Codex for coding help, research, and general workflow stuff."
  - "Probably around $100-$200 a month overall."
  - "Usually whoever manages the subscriptions notices when the bill suddenly goes up."
  - "We have switched plans and stopped using some tools when they did not feel worth the cost anymore."
  - "I would trust it more if it clearly showed the actual usage data behind the recommendations."
  - "Yes, as long as it is anonymized properly and does not include sensitive information."
- Most surprising thing: Abhishek's team uses multiple overlapping tools, but the decision to keep or cancel still comes down to whether the tool feels worth the cost in normal workflow.
- What it changed in the product: Reinforced that the public share page and saved report must be anonymized by default. It also pushed the audit copy toward explaining "why this is worth keeping" for low-savings cases, not only "what to cut."
